# Задание 1. Аннотации

## Задача 1. `@Invoke`

### Текст задачи
Разработать аннотацию `@Invoke` со следующими характеристиками:
- целью может быть только **метод**;
- аннотация доступна **во время исполнения программы**;
- аннотация **не имеет свойств**.

Необходимо создать класс, содержащий несколько методов, и проаннотировать хотя бы один из них аннотацией `@Invoke`. Также требуется реализовать обработчик через **Reflection API**, который находит методы, отмеченные аннотацией `@Invoke`, и вызывает их автоматически.

### Алгоритм решения
1. Создана аннотация `@Invoke`.
2. В аннотации указаны:
   - `@Target(ElementType.METHOD)`
   - `@Retention(RetentionPolicy.RUNTIME)`
3. Создан демонстрационный класс с несколькими методами, один или несколько из которых отмечены `@Invoke`.
4. Реализован обработчик, который:
   - получает список методов класса через Reflection API;
   - проверяет наличие аннотации `@Invoke`;
   - автоматически вызывает найденные методы.
5. В `main` показана работа обработчика и автоматический вызов аннотированных методов.

## Задача 2. `@Default`

### Текст задачи
Разработать аннотацию `@Default` со следующими характеристиками:
- целью может быть **тип** или **поле**;
- аннотация доступна **во время исполнения программы**;
- имеет обязательное свойство `value` типа `Class`.

Необходимо проаннотировать какой-либо класс данной аннотацией, указав тип по умолчанию, и написать обработчик, который выводит имя указанного класса по умолчанию.

### Алгоритм решения
1. Создана аннотация `@Default`.
2. В аннотации указаны:
   - `@Target({ElementType.TYPE, ElementType.FIELD})`
   - `@Retention(RetentionPolicy.RUNTIME)`
   - обязательное свойство `Class<?> value()`
3. Аннотация установлена на класс или поле с указанием типа по умолчанию.
4. Реализован обработчик, который через Reflection:
   - считывает экземпляр аннотации;
   - извлекает значение свойства `value`;
   - выводит имя класса, указанного по умолчанию.
5. В `main` демонстрируется чтение аннотации и вывод результата.

## Задача 3. `@ToString`

### Текст задачи
Разработать аннотацию `@ToString` со следующими характеристиками:
- целью может быть **тип** или **поле**;
- аннотация доступна **во время исполнения программы**;
- имеет необязательное свойство `value` с двумя вариантами значений: `YES` или `NO`;
- значение свойства по умолчанию — `YES`.

Необходимо проаннотировать класс аннотацией `@ToString`, а одно из полей — `@ToString(Mode.NO)`. После этого нужно создать метод, который формирует строковое представление объекта, учитывая только те поля, где `@ToString` имеет значение `YES`.

### Алгоритм решения
1. Создана аннотация `@ToString` и перечисление `Mode { YES, NO }`.
2. Для аннотации заданы:
   - `@Target({ElementType.TYPE, ElementType.FIELD})`
   - `@Retention(RetentionPolicy.RUNTIME)`
   - свойство `value()` со значением по умолчанию `Mode.YES`
3. Создан класс, помеченный аннотацией `@ToString`.
4. Одно или несколько полей класса помечены как `@ToString(Mode.NO)`.
5. Реализован метод формирования строкового представления объекта:
   - перебираются поля через Reflection API;
   - исключаются поля с режимом `NO`;
   - остальные добавляются в результирующую строку.
6. В `main` выводится строковое представление объекта по правилам аннотации.

## Задача 4. `@Validate`

### Текст задачи
Разработать аннотацию `@Validate` со следующими характеристиками:
- целью может быть **тип** или **аннотация**;
- аннотация доступна **во время исполнения программы**;
- имеет обязательное свойство `value` типа `Class[]`.

Необходимо проаннотировать класс аннотацией `@Validate`, передав список типов для проверки. Требуется реализовать обработчик, который выводит, какие классы указаны в аннотации.

### Алгоритм решения
1. Создана аннотация `@Validate`.
2. В аннотации указаны:
   - `@Target({ElementType.TYPE, ElementType.ANNOTATION_TYPE})`
   - `@Retention(RetentionPolicy.RUNTIME)`
   - свойство `Class<?>[] value()`
3. Создан класс с аннотацией `@Validate`, содержащей массив типов.
4. Реализован обработчик, который:
   - получает аннотацию у класса;
   - извлекает массив классов;
   - выводит список типов, переданных в аннотации.
5. При необходимости добавлена проверка на пустой массив.
6. В `main` показана работа обработчика на демонстрационном классе.

## Задача 5. `@Two`

   - `@Target(ElementType.TYPE)`
   - `@Retention(RetentionPolicy.RUNTIME)`
   - свойства `String first()` и `int second()`
3. Создан демонстрационный класс, помеченный `@Two`.
4. Реализован обработчик, который через Reflection:
   - получает экземпляр аннотации;
   - считывает значения `first` и `second`;
   - выводит их пользователю.
5. В `main` демонстрируется корректное считывание свойств аннотации.

## Задача 6. `@Cache`

### Текст задачи
Разработать аннотацию `@Cache` со следующими характеристиками:
- целью может быть только **тип**;
- аннотация доступна **во время исполнения программы**;
- имеет необязательное свойство `value` типа `String[]`;
- значение свойства по умолчанию — пустой массив.

Необходимо проаннотировать класс аннотацией `@Cache`, указав несколько кешируемых областей. Также требуется создать обработчик, который выводит список всех кешируемых областей или сообщение о том, что список пуст.

### Алгоритм решения
1. Создана аннотация `@Cache`.
2. В аннотации указаны:
   - `@Target(ElementType.TYPE)`
   - `@Retention(RetentionPolicy.RUNTIME)`
   - свойство `String[] value() default {}`
3. Создан класс, помеченный аннотацией `@Cache`.
4. В аннотации указаны имена кешируемых областей, например: `users`, `orders`, `products`.
5. Реализован обработчик, который:
   - извлекает значения аннотации;
   - выводит список областей кеширования;
   - при пустом массиве выводит отдельное сообщение.
6. В `main` демонстрируется чтение списка кешей.

# Задание 2. Тестирование

## Задача 4. Тестирование `@Validate`

### Текст задачи
Написать тест с использованием **JUnit**, который проверяет корректность работы механизма валидации классов, отмеченных аннотацией `@Validate`.

Требуется:
- создать класс с аннотацией `@Validate`, указывающей массив типов для проверки;
- использовать тест, который вызывает обработчик и проверяет, что список классов, переданный в аннотации, корректно извлекается;
- проверить, что при передаче пустого массива выбрасывается `IllegalArgumentException`;
- использовать `assertThrows` из JUnit 5.

### Алгоритм решения
1. Создан тестовый класс с аннотацией `@Validate`.
2. Написан JUnit-тест, который:
   - вызывает обработчик чтения аннотации;
   - получает массив классов;
   - сравнивает извлеченные значения с ожидаемыми.
3. Добавлен отдельный тест для случая пустого массива.
4. В тесте используется `assertThrows(IllegalArgumentException.class, ...)`.
5. Проверяется корректность обработки как валидного, так и ошибочного сценария.

## Задача 2. Тестирование `@Invoke`

### Текст задачи
Создать тест с использованием **JUnit**, который проверяет корректность вызова методов, отмеченных аннотацией `@Invoke`.

Необходимо:
- использовать Reflection API для поиска методов с аннотацией;
- убедиться, что метод действительно выполняется без исключений;
- проверить, что возвращаемое значение или побочный эффект соответствует ожиданиям;
- использовать аннотацию `@BeforeEach` для подготовки тестируемого экземпляра класса.

### Алгоритм решения
1. Создан класс с методом, помеченным аннотацией `@Invoke`.
2. Перед каждым тестом через `@BeforeEach` создается новый экземпляр тестируемого объекта.
3. В тесте через Reflection ищутся методы с аннотацией `@Invoke`.
4. Найденные методы вызываются автоматически.
5. Проверяется результат вызова:
   - изменение флага;
   - изменение состояния объекта;
   - либо ожидаемое возвращаемое значение.
6. Тест подтверждает, что механизм автоматического вызова аннотированных методов работает корректно.
